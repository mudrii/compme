**Carry into A1 (key A0 findings so far):**
- **macOS-FFI signature drift vs the draft probe code (all fixed to compile against the pinned versions; do NOT bump versions):**
  - accessibility-sys 0.2.0: `AXIsProcessTrusted` and `AXValueGetValue` return `bool` (use `!trusted` / the bool directly, not `== 0` / `!= 0`). All other AX symbols/sigs matched the draft. `kAXValueType*` are `u32` (`AXValueType`); AX attribute-name consts are `&str`; `CFRange`/`CFTypeRef` are re-exported from `core_foundation::base`.
  - core-graphics 0.25.0 `CGEventTap`: callback returns `CallbackResult` (`Keep`/`Drop`/`Replace`), NOT `Option<CGEvent>`; `mach_port` is private → `tap.mach_port()`; `CFRunLoop::add_source` is a SAFE fn now; keycode field key is `EventField::KEYBOARD_EVENT_KEYCODE` (a const on the `EventField` struct), NOT on `CGEventField` (which is `type = u32`).
  - core-foundation 0.10.1: `kCFRunLoopCommonModes` is re-exported via `core_foundation::runloop` (from `-sys`) but is an extern `static`, so reading it requires `unsafe`.
  - objc2 0.6.4 / objc2-app-kit 0.3.2: `T::alloc(mtm)` requires `objc2::MainThreadOnly` in scope. **NSPanel** init (`initWithContentRect_styleMask_backing_defer`) + the window/control setters are **safe** fns (NSWindow's init is `unsafe`, but NSPanel overrides it safe) → no `unsafe` blocks for the overlay. AppKit types are feature-gated but all needed ones (incl. `NSGraphics`, which gates the panel init) are in the crate's default features, so no `Cargo.toml` feature changes were required. Confirmed-good draft calls: `setLevel(101: NSWindowLevel=isize)`, `NSColor::{clearColor, colorWithWhite_alpha}`, `NSTextField::labelWithString`, `NSScreen::mainScreen(mtm)`, `NSApplicationActivationPolicy::Accessory`, `NSBackingStoreType::Buffered`, `NSWindowStyleMask::{Borderless,NonactivatingPanel}` (bitflags).
- **Coordinate conversion:** AX returns top-left-origin `CGRect`; Cocoa panel placement needs the bottom-left flip via `screen_h - y - h` (lib `ax_to_cocoa_origin`); `NSScreen::frame()` gives `screen_h` (objc2_core_foundation `CGRect`, CGFloat=f64). Retina/multi-monitor pixel-vs-point offset still to be quantified in the manual run.
- **Seam shape validated end-to-end:** `BoundsSource`/`FieldSource`/`Completer` (the A1 `PlatformAdapter` prototype) compile against real AX + llama with the bins as pure glue — the tested lib owns ladder/context/pipeline/coord logic. `Special::Tokenize`/`token_to_str` are deprecated in llama-cpp-2 0.1.146 (`token_to_piece` is the replacement) — adopt in A1.
